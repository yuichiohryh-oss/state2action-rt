# AGENTS.md

このリポジトリで作業する自動実装エージェント（Codex等）向けの共通ルール。

## 基本方針
- 変更は「小さく、独立し、レビューしやすく」。
- 既存のAPI/スキーマ/CLIの互換性を優先。破壊的変更は避ける（必要な場合は別PRで相談）。

## ブランチ運用
- 作業開始時に必ず `main` を最新化する。
- 常に最新 `main` から feature ブランチを作成する。
- ブランチ名例: `feat/ocr-ui-predictor`, `fix/readme-bom`

## PR作成（原則必須）
- エラー等が無く、要件を満たした場合は **必ずPRを作成**する。
- PR本文には以下を含める：
  - 変更概要
  - 動作確認コマンド
  - 既知の制約 / 今後の拡張点（あれば）

## 実行結果の返答
- 実行結果は、ChatGPTのチャットにコピペして確認します。ChatGPTにレビューを依頼する文章にしてください。
- 私が、ChatGPTにコピペしやすいようにしてください。
- 返答には、PRのURLを含めてください。
